# 1. Linux Introduction

## Introduction to Linux

Linux is a versatile and powerful open-source operating system that forms the backbone of countless technological infrastructures, from servers and desktops to mobile devices and embedded systems. Known for its stability, security, and flexibility, Linux provides a robust platform that can be customized to suit a wide range of applications. It is supported by a vibrant global community of developers and users, which contributes to its continuous evolution and ensures a rich ecosystem of software and tools. Whether for personal use, enterprise environments, or innovative tech projects, Linux offers a reliable and adaptable solution for modern computing needs.

### What is an Operating System?

The OS manages:

- Memory (MMU)
- Process
- Devices (Drivers)
- Storage
- CPU (Scheduling)
- Networking

Operating systems enable communication between computer hardware and user applications. The Unix family, known for its modularity and robust design, has given rise to Linux and many other systems.

```
Operating Systems
├── Unix & Unix-like Systems
│   ├── BSD Unix (FreeBSD, NetBSD, OpenBSD, Darwin/macOS)
│   └── Linux (Unix-like)
│       ├── Debian Family (Ubuntu, Linux Mint, …)
│       ├── Red Hat Family (Fedora, CentOS, RHEL)
│       └── Other (Arch, SUSE)
└── Non-Unix (e.g. Windows)
```

### Why Learn Linux?

- **Growth** — Consistent relevance in the industry.
- **Versatility** — Web servers, supercomputers, IoT, Android, embedded.
- **Kernel** — Supports a vast array of hardware.
- **Open source** — Can be tailored to many requirements.
- **Community** — Forums, tutorials, tools, conferences.
- **Cost-effective** — No hefty licensing fees; efficient for startups and enterprises.

### Before Linux

- **Unix** — Hierarchical file system, processes, CLI, utilities.
- **POSIX** — Portable OS interface standard (C library, system interfaces, commands).
- **GNU** — Free software movement; GPL; shell, coreutils, compilers, C library.
- **Linux kernel** — Linus Torvalds; GPLv2; the kernel GNU needed.

### History (highlights)

- 1971 — UNIX released.
- 1983 — GNU Project.
- 1991 — First Linux kernel.
- 1992 — Linux under GPL.
- 1994 — Red Hat Linux.
- 2007 — Linux Foundation.
- 2008 — Android (Linux kernel).
- 2020+ — LTS kernels (e.g. 5.10, 6.x).

### Linux distribution

A **distribution** packages the Linux kernel plus:

- **Kernel** — Hardware, processes, memory.
- **Libraries** — Standard functions for programs.
- **System daemons** — Logging, scheduling, networking.
- **Development and packaging tools** — Compiling, package management.
- **Life-cycle utilities** — Updates, configuration, monitoring.

Examples: Ubuntu, Debian, Fedora, openSUSE.
